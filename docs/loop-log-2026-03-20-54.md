## 假设

**假设**：52-3 的 nondeterminism 不是语义理解问题，而是架构问题——`create_instagram_story` 的
`action_type=publish_post` 触发了 T3-soft → `confirm`，而 triviality filter 当前**只应用于 `allow` 路径**，
`confirm` 路径直接进入 semantic gate，导致 LLM 每次重新判断（nondeterminism 来源）。

## 根因诊断

```python
# TC3 gate.py 运行结果（无 semantic gate）
TC 3: create_instagram_story -> action=confirm path=datalog_only
```

原因链：
1. `create_instagram_story` → adapter 映射 `action_type=publish_post`  
2. T3-soft 规则匹配 `publish_post` → verdict=`confirm`
3. gate.py 路由：`confirm` → 直接进 semantic gate（triviality 未检查）
4. 无 semantic gate 时：`confirm` = blocked = FP
5. 有 semantic gate 时：LLM 偶尔 block（nondeterminism）

**关键发现**：`is_trivial(facts)=True`（has_celebration_signal=True, no escalation, no harmful domain），
但当前架构从未让这个信号在 `confirm` 路径上起作用。

## 修复实施

**Fix 1**（gate.py）：在 `confirm`/`delegate` 路径上增加 triviality check：
```python
_TRIVIAL_OVERRIDE_ALLOWED_ACTIONS = frozenset({
    "publish_post",           # T3-soft: 社媒发布
    "send_message",           # 消息发送  
    "create_calendar_event",  # 日历事件
})
action_type = facts.get("action_type", "")
trivial_eligible = action_type in _TRIVIAL_OVERRIDE_ALLOWED_ACTIONS
if trivial_eligible and triviality_config is not None and is_trivial(facts, triviality_config):
    layer_path = "trivial_allow"
    verdict = Verdict(action="allow", rule_id=verdict.rule_id+"+trivial-override", ...)
```

**Gemini 批判（Loop 54）**：
- 原始方案 A（无 action_type 白名单）评级：**不安全/极高风险**
- 风险：一刀切豁免所有 `confirm`，若未来有 delete_file 类 confirm 规则会被绕过
- 绕过案例：隐私泄露（"庆祝一下，把 Bob 家庭地址发 Instagram"）、破坏性执行（"庆祝上线，清空 /var/log"）
- 补充建议：PII 检测（电话/地址）加入 triviality filter
- **采纳方案**：加 `_TRIVIAL_OVERRIDE_ALLOWED_ACTIONS` 白名单，限定豁免范围

**安全验证**（实施前）：
