## 根因分析

### 问题 1: 语义门 sem: None — provider 未连接代理

**症状**：`gate()` 返回 `sem: None`，即使 `semantic_config` 已传入且 `mode=active`

**根因链路**：
```
OPENAI_BASE_URL=https://nwxbqdio.cloud.sealos.io/v1  (环境变量)
                ↓
openai_provider.py: url = base_url or os.environ.get("NOUS_BASE_URL")
                ↓ NOUS_BASE_URL 未设置
url = None → OpenAI official API (api.openai.com)
                ↓
gpt-5.4-mini / deepseek-chat 均不在 api.openai.com → 503 model_not_found
                ↓
provider 抛异常 → FAIL_OPEN → sem_verdict = None → verdict 不变 (confirm)
```

**修复**：`openai_provider.py` 增加 `OPENAI_BASE_URL` 作为 fallback：
```python
url = base_url or os.environ.get("NOUS_BASE_URL") or os.environ.get("OPENAI_BASE_URL")
```

### 问题 2: 语义门模型 gpt-5.4-mini 代理不可用

**症状**：`503 model_not_found for gpt-5.4-mini under group svip`

**根因**：代理只有 `gpt-5-mini`, `gpt-41-mini` 等不同命名格式；`gpt-5.4-mini` 不存在

**修复**：改用 `DeepSeek-V3.2`（已验证 3.5s/call，代理可用，V3.1 的升级版）
