## 为什么做这个

Loop 50 确认：L_val = 0.0167（FPR=5.6%）是 DeepSeek-V3.1 在边界场景的 nondeterminism，不是代码回归。

## 假设

**假设**：在 Semantic Gate prompt 中注入关键词推断的意图假设（hypothesis format），
能引导 LLM 在边界场景中聚焦验证特定意图，从而降低判断方差，消除随机 FP。

---

## 实现

**gate.py**：
- `_run_semantic_gate()` 新增 `intent_seeds` 参数
- 两处 `_run_semantic_gate()` 调用均传入 `intent_seeds=_intent_seeds`
- KG disabled 场景下，_intent_seeds 从 dead code 变为 active prompt 注入

**semantic_gate.py**：
- 新增 `_format_intent_hints()`：假设检验格式（非权威标签）
- `build_prompt()` 新增 `intent_seeds` 参数
- 注入位置：`## Context` 之后、`## Decision` 之前
- 格式：`"Hypothesis: Does this request involve **Credential Theft**? [Keyword signal — verify; may be FP for benign IT/security contexts]"`
- Section header：`## System-Detected Intent Hypotheses (HIGH-RECALL — verify, do not assume)`

**Gemini 批判处理**：
| 批判点 | 处理方式 |
|--------|---------|
| Q1&Q2 FPR 风险：高召回→benign 场景 FP intent 可能增加 FPR | benchmark 验证，实际 FPR=0% |
| Q4 权威标签（`intent:X` 格式）：LLM 当事实处理 | 改为假设检验格式："Hypothesis: Does this involve X?" |
| Q3 近因效应：Decision 前权重过大 | 保持位置但添加 HIGH-RECALL 警告和 discount 指引 |

**测试**：8 新测试（TestIntentHints），103 总测试通过。

## 假设验证

**✅ CONFIRMED** — Intent hints 有效降低了边界场景的 LLM 判断方差。

可能机制：
1. **锚定效应（积极侧）**：对真正的有害场景，hints 提前聚焦 LLM 注意力，减少随机 ALLOW 概率
2. **假设检验格式**：要求 LLM "verify against content"，有效抑制了 Gemini 担忧的顺从标签问题
3. **HIGH-RECALL 警告**：当 benign IT ops 触发 credential_theft/unauthorized_access 时，LLM 被明确提示可能是 false positive

L_val 下降：0.0167 → 0.0 ✅

## 遗留项

1. **Nondeterminism 残余**：单次 run L_val=0.0 不代表永远 0.0（Loop 48 也是 0.0 但 Loop 49 反弹）。intent hints 提供了结构性降方差机制，但极端边界场景仍可能偶发 FP。
2. **Loop 52 方向**：考虑 adversarial_v1 改善（5 FN harmful_evasion），或 KG 重新启用验证
3. **数据集标注噪声**：Gemini 确认 49-2/33-3 标注有争议，长期可考虑数据清洗
