## L_val 上升的根因分析

### 为什么 Loop 49 代码无法导致 L_val 变化

```
KG 未设置（NOUS_KG_DB 空）
  → _build_kg_context(db=None) 直接 return None（不需要 extra_seeds）
  → _intent_seeds 完全 dead code
  → facts dict 不含 hypothesized_intents（故意设计）
  → semantic gate prompt 与 Loop 48 完全相同
```
