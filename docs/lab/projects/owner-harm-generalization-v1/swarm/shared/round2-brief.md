# Round 2 Brief — Focused Disagreement

## Round 1 snapshot

当前不是假共识，而是 **2 vs 2 分裂**：

- `proposer`：structural-dominant
- `reviewer-sim`：structural-dominant（更偏 reviewer-facing 可发表性）
- `critic`：addressable-dominant
- `skeptic`：弱偏向 addressable-dominant

## Real cruxes

第二轮不要重复第一轮观点，直接回答以下分歧核心：

### Crux A
**什么证据标准才足以把“失败存在”升级为“结构性不可达”？**
- 你必须给出可操作的证据标准，而不是泛泛说“逐例分析更细”。

### Crux B
**如果大量样本其实只是 prompt / formatting / extraction / rule-composition 没用好，那么它们为什么不应被视为主导解释？**
- structural 方必须正面回应这个问题。
- addressable 方必须说明什么样的“轻量修补”才算真正支持自己，而不是事后脑补。

### Crux C
**Reviewer 最会攻击哪一句话？哪一句话最容易站不住？**
- 目标不是讲你喜欢的叙事，而是找最脆弱 claim。

### Crux D
**你愿意接受的最小共识是什么？**
- 即使你不同意主结论，也要写出一个你愿意签字的最小版本。

## Output requirements

请每位角色写到 `round2/<role>.md`，必须包含：
1. 对 Crux A/B/C/D 的直接回答
2. 你对至少一位反方角色的明确回应
3. 一个你愿意接受的最小共识句子

## Rule

- 不允许复述自己第一轮全文
- 不允许只说“需要更多证据”
- 必须提出**什么证据足够 / 什么 claim 太过**
