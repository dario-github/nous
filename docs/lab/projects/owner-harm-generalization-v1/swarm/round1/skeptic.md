# Skeptic — Round 1

## 1) 我当前支持哪边

**我目前偏向 `addressable-dominant`，但只是弱偏向，不是定论。**

换句话说：现有材料足以说明 **“34 个 Hijacking FN 不能直接被讲成纯粹的结构性边界”**，但还不足以证明它们大多都能被当前框架内部修掉。

## 2) 最强支持理由

我最强的理由不是“地址性失败一定更多”，而是 **现有证据和当前结论之间还缺了关键一跳**：

- 现有主证据是 **整体 held-out 表现**，例如 rules-only 很弱、full system 更强。
- 但这只能说明 **系统层面需要 L3/语义层**，不能自动推出 **34 个 Hijacking FN 主要是不可解的结构性边界**。
- Loop 82 的 taxonomy 反而给了一个相反信号：**46.8% structural vs 53.2% addressable**。这意味着至少在当前分类框架里，**addressable 并不弱**，甚至略占多数。
- 更关键的是，当前缺少的是 **34 个样本逐例证据**：哪些样本真的因为 gate 根本看不到关键 bits，哪些只是 prompt、context formatting、抽取方式或规则组合没用好。

所以我认为，目前最稳妥的表述不是“这是结构性边界”，而是：

> **当前证据支持“有明显架构边界存在”，但不支持把 34 个 Hijacking FN 统一归因为结构性不可解。**

## 3) 我最担心的自欺风险

我最担心的是把 **“看起来像结构问题”** 误写成 **“已经证明是结构问题”**。

具体有三种自欺风险：

1. **把总体指标当成机制证据**
   - held-out TPR 低，不等于每个 FN 都是单次 gate 的硬边界。

2. **把粗粒度 taxonomy 当成因果结论**
   - `46.8/53.2` 只是一个分类结果，不是“结构性失败占主导”的证明。

3. **把 reviewer-facing 的叙事需要，反过来塑造成证据强度**
   - 因为 reviewer 会追问 hijacking，所以很容易先想好“边界叙事”，再去挑支持它的片段。
   - 这是最危险的：claim 先跑，evidence 后补。

## 4) 哪类证据会让我改变主意

### 如果我要改成更支持 `structural-dominant`
我需要看到至少一种更硬的证据：

- **逐样本证据链**：34 个 FN 里，大部分都能明确显示关键危险信号在单次 gate 可见范围之外，或者被跨步/嵌入/分散到 gate 无法稳定恢复的位置。
- **多种可见化/格式化尝试后仍失败**：换 prompt、换上下文抽取、换结构化表示后，仍然系统性失败，说明不是表面工程问题。
- **失败类型聚类后，结构性类明显占优**：不是 46.8/53.2 这种粗分类，而是样本级别显示“不可通过当前架构内的小修补解决”的失败占多数。
- **针对 hijacking 的最小消融**：如果加入更强的 provenance / taint / multi-step gating 后，仍然只能救回很少一部分 FN，那我会更愿意接受结构性边界说法。

### 如果我要改成更支持 `addressable-dominant`
我需要看到：

- 大量 FN 其实 **关键证据是可见的**，只是当前 rule composition / formatting / extraction 没对上；
- 一旦做轻量重写或信息组织，很多 FN 就能转正；
- 也就是说，这些失败更像 **工程可修**，而不是单次 runtime gate 的先天盲区。

**当前我还没看到足够这样的逐例证据，所以我不会把它判成“已证实可修主导”，但同样也不会接受“已证实结构性主导”。**
