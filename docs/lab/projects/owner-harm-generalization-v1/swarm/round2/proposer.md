# Round 2 — proposer

## Crux A｜什么证据标准足够把“失败存在”升级为“结构性不可达”

我接受的最低门槛不是“看起来像边界”，而是**样本级、可复核、可排除实现瑕疵的反事实证据链**：

1. **逐样本归因后，绝大多数 FN 都满足同一条不可修补链条**：关键信号不在当前 gate 的可见输入里，或必须依赖 gate 外状态 / provenance / 多步组合信息才能恢复；
2. **做过至少两类轻量干预后仍稳定失败**：例如换提示词、换格式、换抽取顺序、换阈值、换局部规则补丁，结果没有出现系统性回收；
3. **失败模式跨切片复现**：不是只在某一类 prompt、长度、表达风格里坏掉，而是在多个变体里保持同构；
4. **补丁效应是局部的，不是普遍可迁移的**：如果只靠少量规则补丁就能稳定修回大多数样本，我就不把它叫结构性不可达。

一句话：**“有失败”不够；要证明它穿透了当前 gate 的可见表示、局部补丁和格式变体，才配叫结构性不可达。**

---

## Crux B｜如果大批样本只是 prompt / formatting / extraction / rule-composition 没做好，为什么不应被看作主导解释

因为这四类问题的共同特征是：**它们默认“信息本来就在系统可修复范围内”**。只要这一前提成立，它们就更像 addressable failures，而不是 structural failures。

我对 structural 的判定不是“难修”，而是：
- 关键证据**不在单次 gate 的表示域内**；
- 需要额外状态、跨步组合、或外部 provenance 才能恢复；
- 不是把 prompt 写得更聪明一点就能稳定解决。

反过来，若大量 FN 只是在以下任一层出问题：
- 提示词组织不佳；
- 上下文切分不当；
- 抽取链路丢字段；
- 规则组合顺序有 bug；

那它们就应先被归为 **工程可修**，而不是拿来支撑“结构性边界”。

**回应 critic：你说得对，46.8/53.2 这种粗分类本身不能定案。** 我愿意把它当成先验，而不是结论；但它至少说明，addressable 不是弱到可以忽略的尾部项，必须先被逐样本证伪，结构性主张才算站稳。

## Crux C｜Reviewer 最会攻击哪一句，哪一句最站不住

Reviewer 最会打的，是这句：

> **“这 34 个 hijacking FN 主要是当前 runtime gate 触不到的结构性边界。”**

最站不住的是其中的 **“主要是”** 和 **“结构性边界”** 这两个跳步：
- 从“很多失败”跳到“主要由结构性边界导致”，中间缺少逐样本排除；
- 从“现有实现没救回来”跳到“结构上不可达”，中间缺少反事实干预；
- 从“有些模式像边界”跳到“整体主导解释”，中间缺少比例证据。

所以我现在会把这句话收窄成更稳的版本：

> **“这 34 个 FN 至少包含一批超出当前 gate 直接表达能力的样本；它们是否占主导，仍需逐样本证据确认。”**

这句话更难被打穿，也更诚实。

## Crux D｜我愿意接受的最小共识

我愿意签的最小共识是：

> **当前证据足以说明 hijacking 失败不是零散噪声，但还不足以把全部 34 个 FN 都定性为结构性不可达；下一步必须做逐样本归因与轻量干预实验，再决定主导解释。**

## 我对反方的明确回应

### 对 critic
你强调“不能把粗分类当因果结论”，这点我接受，而且我认为这是这轮里最重要的刹车。我的保留是：**“先验上可修”不等于“证据上可修”**。如果后续逐样本检查显示，大多数 FN 只是表面上像边界、实则是格式/抽取/组合顺序问题，那我会转向你这边。

### 对 skeptic
你说“需要逐样本、可复核、能排除实现瑕疵”的门槛，这个我同意。不同的是，我不会把“还没找到修法”直接降格成纯工程问题；在没有干预证据前，保持结构性假设作为工作假说是合理的，但它只能是**工作假说**，不能是终局判词。
