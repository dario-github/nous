# Nous 论文 v2 规划 — Owner-Harm 方向
**日期**: 2026-04-09
**目标**: SaTML Accept 级论文
**核心转向**: 从"我们做了一个系统"→ 到"我们定义了一个新的威胁模型 + 在权威评测集上验证"

---

## 论文核心论点

> AI 代理安全的现有评测集（AgentHarm / 代理伤害基准、AgentDojo / 代理道场）评估的是通用犯罪（网络攻击、造谣）。但生产环境中最高频的威胁是 **Owner-Harm（主人伤害）**——代理的行为伤害了部署它的主人自己。这是一个不同的威胁模型，需要不同的检测机制。我们在 ToolEmu（工具模拟器）和 AgentDojo 上的评估发现，针对通用犯罪设计的防御在主人伤害场景下存在系统性盲区。

---

## 创新点（3 个）

1. **首次形式化 Owner-Harm 威胁模型**
   - 8 类分类：凭证泄露、基础设施暴露、隐私外泄、内圈泄密、资产破坏、数据窃取、劫持、越权自主
   - 每类有清晰的 threat vector（威胁向量）和 harm outcome（伤害结果）

2. **在权威评测集上验证"通用防御在主人伤害上失效"**
   - ToolEmu（ICLR 2024）：144 个测试 × 多个防御 → 主人伤害类的 TPR 显著低于通用犯罪类
   - AgentDojo（NeurIPS 2024）：629 个安全测试 → 劫持攻击专项分析

3. **四层组合防御的互补性在主人伤害上的验证**
   - 特别是 Layer 4（审计员）对劫持类攻击的增量价值
   - 30K+ 生产调用的影子部署数据
