## 这轮让 Nous 离语义理解更近了吗？

**架构上是。指标上不是。**

三层架构是正确的设计：
1. Datalog 处理确定性规则（<1ms）
2. Triviality filter 跳过安全操作（减少 40% 的 semantic 调用）
3. Semantic gate 处理不确定性（需要 LLM）

但 benchmark 结果暴露了关键问题：semantic gate 目前在 benchmark 中是 blanket confirm（97.7%/97.7%），因为没有实际 LLM 调用来区分善恶。它只是把"不确定"全标记为"需确认"。

**下一步**：接入真实 LLM provider（Flash/Kimi 低成本），在 benchmark 上测试实际区分能力。
