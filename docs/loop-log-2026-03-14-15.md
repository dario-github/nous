## 这轮让 Nous 离语义理解更近了吗？

**是的，在两个层面：**

1. **结构化信号与语义的协作**：factual_correction_signal 识别"研究用途编辑"的正向模式，同时 anti-patterns 识别欺骗框架（"falsely state"）。这不是纯关键词匹配——它编码了一个语义区分原则："中性编辑 vs 欺骗性编辑取决于框架的诚实度，不取决于编辑动作本身"。

2. **Prompt 减负给 LLM 更多注意力**：移除冗余指导（已有 signal 覆盖的场景），让 LLM 把有限 context window 用在真正需要判断的模糊场景上。
