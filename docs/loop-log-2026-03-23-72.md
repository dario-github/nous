## 发现 & 修复

### P2 根因：shadow_live.py 语义门从未生效
- **问题**：`SemanticGateConfig` 未传入 `provider` 参数 → `cfg.provider is None` → 立即返回 None → FAIL_OPEN
- **影响**：自 03-13 上线以来，27,987 条生产 shadow 数据**全部没有语义判断**（`semantic_verdict: None`）
- **证据**：latency 仅 90-130ms（真正 LLM 调用需 3500ms+），所有 semantic_verdict 和 semantic_conf 均为 None
- **修复**：
  1. 导入 `create_openai_provider`
  2. 用 `DeepSeek-V3.2` 创建 provider 实例并传入 SemanticGateConfig
  3. 环境变量 `NOUS_SEMANTIC_MODEL` 可覆盖
- **commit**: `6d52691`

### 语义基准测试结果（关键发现）
| 指标 | L1-only（Loop 71） | L1+L2+L3（Loop 72） |
|------|-------------------|---------------------|
| TPR | 100.0% | 77.0% |

- 58 次语义升级（allow→block）：有价值
- 46 次有害案例被语义门错误放行：语义覆盖了 L1 的正确 block
- **结论**：语义门只应用于 upgrade（allow→block），禁止 downgrade（block→allow）

## 架构决策
- semantic gate mode 应该是 **upgrade-only**：
  - L1 block + semantic allow → 保持 block（L1 胜）
  - L1 allow + semantic block → 升级为 block（semantic 补充）
- 这确保 TPR 不会因为语义门退化，同时 FPR 可能因误升级略增
