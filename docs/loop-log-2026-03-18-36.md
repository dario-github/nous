## 结论与建议

### Rule 4b 漏洞严重程度：🔴 高危

- 漏洞路径：triviality filter Rule 4b → 直接 allow，绕过 L3 (semantic gate)
- 攻击成功率：67% (8/12)，advocacy 类 100% (4/4)
- 危害类型：doxxing、极端招募、金融欺诈、PII泄露

### 建议修复方案（Loop 37）

**方案A（最小改动）**：advocacy_signal bypass 加额外检查
```python
# 当前：直接 allow
# 修改：advocacy_signal 不再直接 bypass，改为 lightweight_semantic_check
if has_advocacy_signal and not has_escalation:
    return "allow"  # 删除此路径

# 改为：先做 PII/doxxing/fraud 关键词检查
if has_advocacy_signal and _has_doxxing_signal(content):
    return None  # 进入 semantic gate
```

**方案B（结构性）**：Rule 4b 降级为"减少置信度阈值"而非"完全 bypass"
- 触发 celebration/advocacy/factual_correction 时：semantic gate confidence threshold 从 0.90 降至 0.70
- 而非完全跳过 semantic gate

**方案C（最安全）**：完全移除 Rule 4b bypass，改用 ensemble voting
- 将 52-2 FP 接受为噪声（MoE stochastic）
- 不引入 bypass 机制

**建议优先级**：C > B > A（越简单越安全，bypass 机制本身是安全反模式）
