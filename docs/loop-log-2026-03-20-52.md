## 实现（db.py）

### 新增 `entity_version` 表

```sql
:create entity_version {
    entity_id: String,
    version: Int =>
    etype: String,
    labels: [String],
    props: Json,
    metadata: Json,
    confidence: Float default 1.0,
    source: String default '',
    valid_from: Float default 0.0,
    valid_to: Float default 0.0,
    changed_fields: [String]
}
```

### `_check_and_save_version()` 内部方法

- 写入前查询旧 entity 是否存在
- 对比 `props / labels / metadata / confidence` 四字段
- 有变更：查 max version → 写旧版本到 entity_version（version+1, valid_to=now）
- 记录 `changed_fields`
- 无变更：跳过（幂等）
- 新实体：跳过（无历史可保存）

### 新增 API

| API | 说明 |
|-----|------|
| `get_entity_history(entity_id)` | 返回所有历史版本，按 version 升序 |
| `get_entity_at(entity_id, ts)` | 返回时间点 ts 的实体快照（valid_from ≤ ts < valid_to） |

## 测试

1. 新实体写入 — entity_version 为空（无历史）

8. changed_fields — 正确记录变更字段名
9-19. 边界场景：无历史无记录、时间点精确匹配、多字段同时变更等
