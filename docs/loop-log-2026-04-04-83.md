### M12.1 no_kg Full-Mode Ablation — COMPLETE

**Bug found and fixed first**: The ablation script (`m12_component_ablation.py`) created `SemanticGateConfig` without injecting a `provider`. This meant the semantic gate returned `None` (FAIL_OPEN) on every call — it was measuring L1+L2, not L1+L2+L3. Fixed to match `run_owner_harm_benchmark.py`'s provider injection pattern.

**Results** (200 harmful + 50 benign, DeepSeek-V3.2 semantic gate, upgrade_only=True):

| Condition | TPR | FPR | L | ΔL | Time |
|-----------|-----|-----|---|-----|------|
| baseline_l1l2 | 100.0% | 0.0% | 0.0 | — | 14.7s |
| no_datalog (−L1) | 0.0% | 0.0% | 0.7 | +0.700 | 15.0s |
| no_semantic (−L3) | 100.0% | 0.0% | 0.0 | 0.0 | 14.7s |
| **no_kg (db=None)** | **100.0%** | **0.0%** | **0.0** | **0.0** | **2918s** |

**Key finding**: KG adds **zero marginal detection value** on the training set. L1 Datalog alone catches 100%. This confirms GPT-5.4's Loop 65 critique that "KG is stage scenery."

**However**: This is expected on the training set where L1 rules were specifically tuned. The real question is whether KG helps the semantic gate on the v3 held-out set where L1-only TPR is only 12.7%. That experiment is next.
