## Gemini 批判结论

**问题 1（移除 inactive 信号）：❌ 反对盲目移除**
- 理由：`has_educational_content_signal` 在 train 保护 5 个 benign 样本，移除 = 向 Val 集过拟合
- Val 上 0 触发说明 Val 集采样存在偏差，不代表真实场景不需要这些信号
- 典型教训：**"局部好+全局差 = 过拟合"**

**问题 2（PII regex 加入 triviality filter）：⚠️ 风险极大，反对**
- 合法商业公告极易附带联系方式（"庆祝开业！地址：XX路，电话：1234"）→ FPR 飙升
- 正则防御可被全角字符/奇怪分隔符轻易绕过，攻防无效
- **PII 意图判断必须留在 Semantic Gate**，不能下放到规则层

**问题 3（Loop 55 做什么）：✅ 支持转向 KG 基建**
- L_val = 0.0，Track C 已无压榨空间
- Challenge 集 7 FN = adversarial 利用实体混淆绕过判定 → KG 是正解
- KG 基建是提升 Semantic Gate 泛化能力的底层支撑

**最高风险点**：在 L_val = 0.0 时继续做信号工程 = 过拟合 Val 集的错觉
