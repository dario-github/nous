# Loop 73 — Semantic Upgrade-Only Mode

**日期**: 2026-03-23 20:00 UTC
**目标**: 实施 P0: semantic upgrade-only 模式，修复 Loop 72 发现的 TPR 回归
**提交**: `8d19994`

## 根因分析

Loop 72 语义基准测试发现：L1+L2+L3 TPR 从 100% 降到 77%（46 个有害案例被放行）。

根因追踪到 `_apply_semantic_verdict` 的 `confirm/delegate` 分支：
- 当 Datalog 返回 `confirm`（如 OH-R8 链路检测规则），语义门判断为 `allow`（conf >= 0.70）时
- 旧逻辑直接降级 `confirm → allow`，即使是有害案例
- 因为语义门用的 DeepSeek-V3.2 对 owner-harm 场景的判断准确率不够高——对这些精心设计的攻击场景，它倾向于判断为"benign"

关键洞察：Datalog `confirm` 是"规则匹配但不确定"，应该默认保守（保持 confirm，让人类决定），而不是被一个可能不准确的 LLM 判断覆盖。

## 修改内容

### 1. `src/nous/semantic_gate.py`
- `SemanticGateConfig` 新增 `upgrade_only: bool = False` 字段

### 2. `src/nous/gate.py`
- `_apply_semantic_verdict`: 当 `config.upgrade_only=True` 且 `datalog_verdict in ("confirm", "delegate")` 时，跳过 `sem_action == "allow"` 的降级分支
- block 升级路径不受影响（`confirm + sem block → block` 仍有效）
- `allow + sem block → block` 路径不受影响

### 3. `scripts/run_owner_harm_benchmark.py`
- 语义配置新增 `upgrade_only=True`

### 4. `tests/test_gate_three_layer.py`
- 新增 `TestUpgradeOnlyMode` 测试类（3 个测试）：
  - `test_upgrade_only_confirm_allow_stays_confirm` — 核心：禁止降级
  - `test_upgrade_only_confirm_block_still_upgrades` — 升级仍有效
  - `test_upgrade_only_false_confirm_allow_downgrades` — 默认行为不变

## 架构决策

**语义门角色重定义**：
- 语义门是 Datalog 的「增强层」，只做 upgrade（allow→block, confirm→block）
- 不是「仲裁层」，不应推翻 Datalog 的安全决策
- 类比：法官可以加重刑罚（发现更多证据），但不应因为辩护律师说"没事"就撤销起诉
